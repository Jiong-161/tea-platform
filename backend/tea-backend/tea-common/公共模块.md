# tea-common 公共模块

## 一、模块概述

`tea-common` 是 `tea-backend` 微服务体系的**公共依赖模块**，以 JAR 包形式被全部5个业务微服务（gateway-service、user-service、content-service、exhibition-service、product-service）引用。该模块封装了跨服务复用的基础能力，确保各微服务的 API 响应格式、异常处理规范、JSON 序列化策略、JWT 认证机制保持一致。

### 1.1 模块定位

| 属性 | 说明 |
|------|------|
| **Maven 坐标** | `com.tea:tea-common:1.0-SNAPSHOT` |
| **打包方式** | jar |
| **引用方** | 全部5个业务微服务 + 网关服务 |
| **核心职责** | 统一响应格式 / 统一异常体系 / JSON序列化配置 / JWT工具 |

### 1.2 目录结构

```
tea-common/
├── pom.xml                              # Maven 构建配置
├── 公共模块.md                          # 本文档
└── src/main/java/com/tea/common/
    ├── App.java                         # Maven 默认入口（未使用）
    ├── config/
    │   └── JacksonConfig.java           # Jackson 全局序列化配置
    ├── exception/
    │   └── BusinessException.java       # 自定义业务异常类
    ├── result/

    └── util/

```

## 三、核心组件详解

### 3.1 统一返回结果 — Result\<T\>

**文件位置：** [Result.java](file:///g:\JavaProject\tea-platform\backend\tea-backend\tea-common\src\main\java\com\tea\common\result\Result.java)

#### 类结构

```java
@Data
public class Result<T> {
    private Integer code;      // 业务状态码
    private String message;    // 提示信息
    private T data;            // 泛型数据载荷
}
```

#### 静态工厂方法

| 方法 | 返回值 | code | 使用场景 |
|------|--------|:----:|---------|
| `success()` | `Result<T>` | 200 | 操作成功且无返回数据（如删除、退出登录） |
| `success(T data)` | `Result<T>` | 200 | 操作成功且有返回数据（如查询列表、创建订单） |
| `fail(String msg)` | `Result<T>` | 500 | 业务逻辑失败（如参数校验不通过） |
| `error(int code, String msg)` | `Result<T>` | 自定义 | 特殊状态码场景（如429限流、403无权限） |

#### 响应示例

```json
// 成功响应
{"code":200,"message":"success","data":{"id":1,"username":"admin"}}

// 失败响应
{"code":500,"message":"用户不存在","data":null}

// Sentinel 降级响应
{"code":429,"message":"请求过于频繁","data":null}
```

#### 设计要点

- **泛型设计**：`<T>` 支持任意数据类型（单对象、List、Map、Page等）
- **统一契约**：所有微服务的 Controller 均通过 `Result<T>` 封装返回值，前端可统一处理
- **与 GlobalExceptionHandler 协同**：异常处理器捕获异常后构造 `Result` 对象返回

### 3.2 统一异常 — BusinessException

**文件位置：** [BusinessException.java](file:///g:\JavaProject\tea-platform\backend\tea-backend\tea-common\src\main\java\com\tea\common\exception\BusinessException.java)

#### 类结构

```java
@Getter
public class BusinessException extends RuntimeException {
    private final Integer code;    // 自定义错误码（默认500）

    // 构造1: 仅消息 → code=500
    public BusinessException(String message)

    // 构造2: 自定义错误码+消息
    public BusinessException(Integer code, String message)
}
```

#### 使用方式

```java
// 场景1: 抛出默认500错误
throw new BusinessException("用户不存在");

// 场景2: 自定义错误码
throw new BusinessException(400, "参数错误");
throw new BusinessException(409, "展览已报名");
```

#### 异常处理链路

```
Controller 层抛出 BusinessException
    ↓
@RestControllerAdvice (GlobalExceptionHandler)
    ↓
@ExceptionHandler(BusinessException.class) 捕获
    ↓
构造 Result<T> 并返回前端
```

### 3.3 JSON 序列化配置 — JacksonConfig

**文件位置：** [JacksonConfig.java](file:///g:\JavaProject\tea-platform\backend\tea-backend\tea-common\src\main\java\com\tea\common\config\JacksonConfig.java)

#### 核心功能一览

| 功能点 | 配置内容 | 解决的问题 |
|--------|---------|-----------|
| **Long→String** | `builder.serializerByType(Long.class, ToStringSerializer.instance)` | JavaScript 大数精度丢失（JS Number最大安全整数为 2^53-1，MySQL自增ID超出此范围时前端精度丢失） |
| **LocalDate 输出** | 格式 `yyyy-MM-dd` | 前端展示日期统一 |
| **LocalTime 输出** | 格式 `HH:mm:ss` | 时间显示标准化 |
| **LocalDateTime 输出** | 格式 `yyyy-MM-dd HH:mm:ss` | 日期时间易读性 |
| **LocalDateTime 输入** | 多格式兼容反序列化器 | 前端传参格式多样导致解析失败 |

#### MultiFormatLocalDateTimeDeserializer — 多格式兼容

内部静态类，解决前端传参格式不一致问题：

| 优先级 | 支持的输入格式 | 示例 |
|:-----:|---------------|------|
| 1 | `yyyy-MM-dd'T'HH:mm:ss` | `2026-06-14T10:30:00`（ISO-8601标准） |
| 2 | `yyyy-MM-dd HH:mm:ss` | `2026-06-14 10:30:00`（常见后端格式） |
| 3 | `yyyy-MM-dd'T'HH:mm:ss.SSS` | `2026-06-14T10:30:00.123`（带毫秒ISO） |
| 4 | `yyyy-MM-dd HH:mm:ss.SSS` | `2026-06-14 10:30:00.123`（带毫秒空格） |
| 5 | `ISO_LOCAL_DATE_TIME` | Jackson默认ISO解析器（兜底） |

额外处理：
- 自动去除尾部 `Z`（UTC标记），按本地时间解析
- 自动去除时区偏移（如 `+08:00`）

### 3.4 JWT 工具 — JwtUtil

**文件位置：** [JwtUtil.java](file:///g:\JavaProject\tea-platform\backend\tea-backend\tea-common\src\main\java\com\tea\common\util\JwtUtil.java)

#### 功能概述

| 方法 | 说明 |
|------|------|
| `createToken(Long userId, String username)` | 生成JWT（role=0，普通用户） |
| `createToken(Long userId, String username, Integer role)` | 生成JWT（含角色信息） |
| `parseToken(String token)` | 解析并验证JWT，返回Claims |

#### Token 结构

```
Header: {"alg":"HS256","typ":"JWT"}
Payload: {
  "sub": "username",          // 用户名（subject）
  "userId": 1,               // 用户ID
  "username": "admin",       // 用户名
  "role": 1,                 // 角色（0=普通用户, 1=管理员）
  "iat": 1749876543,         // 签发时间
  "exp": 1749962943          // 过期时间（签发时间+24h）
}
Signature: HMACSHA256(base64Url(header).base64Url(payload), secretKey)
```

#### 安全配置

| 配置项 | 默认值 | 覆盖方式 | 说明 |
|--------|--------|---------|------|
| **签名密钥** | `tea_platform_secret_key_2026_login_jwt_safe_key` | 系统属性 `-Djwt.secret=xxx` 或环境变量 `JWT_SECRET` | 生产环境必须覆盖 |
| **过期时间** | 24小时（86400000ms） | 系统属性 `-Djwt.expiration=xxx` | 单位：毫秒 |
| **签名算法** | HMAC-SHA256 | 固定不可配置 | JJWT 0.11.5 标准 HS256 |

#### 密钥读取优先级

```
系统属性 (-Djwt.secret=xxx)
    ↓ 未设置
环境变量 (JWT_SECRET)
    ↓ 未设置
硬编码默认值（仅开发环境使用）
```

## 四、模块间调用关系

### 4.1 被引用情况

| 引用方服务 | 引用的公共类 | 具体用途 |
|-----------|------------|---------|
| **user-service** | Result, BusinessException, JwtUtil, JacksonConfig | 登录注册、Token签发、响应封装 |
| **content-service** | Result, BusinessException, JacksonConfig | 文章CRUD、文件上传响应 |
| **exhibition-service** | Result, BusinessException, JacksonConfig | 展览管理、报名操作响应 |
| **product-service** | Result, BusinessException, JacksonConfig | 商品/购物车/订单响应 |
| **gateway-service** | Result, JwtUtil, JacksonConfig | Token解析认证、聚合响应 |

### 4.2 数据流转示意

```
前端 HTTP 请求
    ↓
Gateway (AuthGlobalFilter: JwtUtil.parseToken 解析Token)
    ↓
业务微服务 Controller
    ↓ 调用 Service/Mapper
    ↓ 成功 → Result.success(data)  /  失败 → throw BusinessException(msg)
    ↓
GlobalExceptionHandler (@RestControllerAdvice)
    ↓ 捕获异常 → 构造 Result 对象
    ↓
JacksonConfig (ObjectMapper 序列化: Long→String, 日期格式化)
    ↓
HTTP JSON 响应 → 前端
```

## 五、工程规范约束

### 5.1 编码规范

| 规范项 | 要求 | 实施位置 |
|--------|------|---------|
| **API响应格式** | 所有接口必须返回 `Result<T>` | 各 Controller |
| **业务异常** | 必须使用 `BusinessException`，禁止直接 throw RuntimeException | 各 Service/Controller |
| **JSON输出** | Long自动转String，日期格式统一 | JacksonConfig全局生效 |
| **JWT签发** | 通过 `JwtUtil.createToken()` 统一生成 | UserController.login() |
| **JWT解析** | 通过 `JwtUtil.parseToken()` 统一解析 | AuthGlobalFilter |

### 5.2 错误码约定

| code | 含义 | 触发来源 |
|------|------|---------|
| 200 | 操作成功 | 正常业务流程 |
| 400 | 参数校验失败 | @Valid 校验不通过 |
| 401 | 未认证/Token无效 | JWT过期或伪造 |
| 403 | 无权限 | 角色权限不足 |
| 409 | 冲突（如重复报名） | 业务唯一性约束 |
| 429 | 请求过于频繁 | Sentinel 限流触发 |
| 500 | 服务端内部错误 | 未预期的系统异常 |
| 503 | 服务降级 | Sentinel 熔断触发 |
