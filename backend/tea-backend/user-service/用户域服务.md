# user-service 用户域服务

## 一、模块概述

`user-service` 是茶文化平台的**用户域微服务**，负责用户全生命周期管理，包括注册登录认证、个人信息维护、收货地址管理、以及管理员后台的用户管理功能。作为系统的核心身份服务，承担 JWT Token 签发与 Redis 会话管理的职责。

### 1.1 模块定位

| 属性 | 说明 |
|------|------|
| **Maven 坐标** | `com.tea:user-service:1.0-SNAPSHOT` |
| **服务端口** | 20001 |
| **Nacos 注册名** | `user-service` |
| **Sentinel Agent端口** | 8720 |
| **数据库** | tea_platform（tea_user / tea_address 两张表） |
| **Redis 用途** | JWT Token 会话缓存（TTL=24h） |

## 五、核心业务流程

### 5.1 用户注册流程

```
POST /user/register
    ↓ @Valid 校验 RegisterDTO (username 4~20位, password 6~20位, nickname非空)
    ↓ 查询用户名是否已存在
    ↓ 已存在 → throw BusinessException("用户名已存在")
    ↓ 不存在 → BCryptPasswordEncoder.encode(password) 加密密码
    ↓ 插入 tea_user 表 (role=0, status=1)
    ↓ 返回 Result.success("注册成功")
```

### 5.2 用户登录流程

```
POST /user/login
    ↓ @Valid 校验 LoginDTO
    ↓ 根据 username 查询 User
    ↓ 用户不存在 → BusinessException("用户不存在")
    ↓ status=0 → BusinessException("账户已被禁用")
    ↓ BCryptPasswordEncoder.matches(rawPassword, encodedPassword)
    ↓ 密码错误 → BusinessException("密码错误")
    ↓ JwtUtil.createToken(userId, username, role) 生成JWT
    ↓ Redis写入 key="login:{token}" value=userId TTL=24h
    ↓ password字段置null脱敏
    ↓ 返回 Result.success(token)
```

**Sentinel 保护**：
- 资源名：`userLogin`
- blockHandler → 返回 `{code:429, "登录请求过于频繁"}`
- fallback → 返回 `{code:503, "登录服务暂不可用"}`

### 5.3 修改密码流程

```
POST /user/password (Header: Authorization: Bearer <token>)
    ↓ 从 X-User-Id 请求头获取userId
    ↓ 从 X-Auth-Token 请求头获取token
    ↓ @Valid 校验 PasswordDTO (oldPassword/newPassword非空)
    ↓ 查询User并校验旧密码 BCrypt.matches()
    ↓ 旧密码错误 → BusinessException("原密码不正确")
    ↓ 新密码加密后更新数据库
    ↓ Redis删除 key="login:{token}" （强制重新登录）
    ↓ 返回 Result.success("修改成功")
```

**Sentinel 保护**：
- 资源名：`userUpdatePassword`
- blockHandler → 返回 `{code:429, "操作过于频繁"}`

### 5.4 退出登录流程

```
POST /user/logout (Header: Authorization: Bearer <token>)
    ↓ 从 X-Auth-Token 请求头获取token
    ↓ Redis删除 key="login:{token}"
    ↓ Redis异常时 try-catch 降级（不影响返回）
    ↓ 返回 Result.success("退出成功")
```

### 5.5 收货地址新增流程

```
POST /user/address/add (Header: Authorization: Bearer <token>)
    ↓ 从 X-User-Id 请求头提取 userId
    ↓ @Valid 校验 AddressDTO (7个字段均@NotBlank)
    ↓ isDefault=1 时调用 clearDefault(userId) 将该用户所有地址置为非默认
    ↓ 构造 Address 实体并插入数据库
    ↓ 返回 Result.success("新增地址成功")
```

### 5.6 设置默认地址流程

```
PUT /user/address/default/{id} (Header: Authorization: Bearer <token>)
    ↓ 从 X-User-Id 提取 userId
    ↓ 查询地址记录，校验 address.userId == 当前用户（归属权校验）
    ↓ 归属权不匹配 → BusinessException("无权操作此地址")
    ↓ clearDefault(userId) 清除其他默认标记
    ↓ 将目标地址 isDefault 设为 1
    ↓ 返回 Result.success("设置成功")
```

## 六、Sentinel 流控与熔断配置

### 6.1 受保护资源列表

| 资源名 | 所属方法 | 保护目标 | blockHandler | fallback |
|--------|---------|---------|-------------|----------|
| `userLogin` | login() | 登录防暴力破解 | loginBlockHandler | loginFallback |
| `userRegister` | register() | 防恶意批量注册 | registerBlockHandler | — |
| `userUpdatePassword` | updatePassword() | 密码修改保护 | updatePasswordBlockHandler | — |

### 6.2 推荐规则配置（在 Sentinel Dashboard 中设置）

#### 流控规则

| 资源名 | 阈值类型 | 阈值 | 说明 |
|--------|---------|------|------|
| `userLogin` | QPS | **10** | 同一来源每秒最多10次登录尝试 |
| `userRegister` | QPS | **5** | 每秒最多5次注册请求 |
| `userUpdatePassword` | QPS | **10** | 每秒最多10次密码修改 |

#### 熔断规则（可选）

| 资源名 | 策略 | 阈值 | 窗口时长 | 最小请求数 |
|--------|------|------|---------|-----------|
| `userLogin` | 慢调用比例 | RT>3000ms 占比>60% | 30s | 5 |

### 6.3 全局异常处理分级

| 异常类型 | HTTP状态码 | 日志级别 |
|----------|:---------:|:-------:|
| FlowException（限流） | 429 | WARN |
| DegradeException（熔断） | 503 | ERROR |
| ParamFlowException（热点限流） | 429 | WARN |
| BusinessException（业务异常） | 自定义code或500 | — |
| ExpiredJwtException（Token过期） | 500 | — |
| MethodArgumentNotValidException（校验失败） | 500 | — |
| Exception（兜底） | 500 | ERROR |

## 七、接口测试指南

### 7.1 测试准备

使用 APifox / Postman / curl 进行接口测试，Base URL 为：

```
http://localhost:10010
```

### 7.2 用户注册

```http
POST /user/register
Content-Type: application/json

{
  "username": "testuser",
  "password": "123456",
  "nickname": "测试用户"
}
```

**响应**：
```json
{"code":200,"message":"success","data":"注册成功"}
```

**异常场景**：
- 用户名已存在 → `{"code":500,"message":"用户名已存在","data":null}`
- 用户名过短(<4字符) → `{"code":500,"message":"参数校验失败","data":null}`
- 密码过短(<6字符) → `{"code":500,"message":"参数校验失败","data":null}`

### 7.3 用户登录

```http
POST /user/login
Content-Type: application/json

{
  "username": "testuser",
  "password": "123456"
}
```

**响应**（data 为 JWT Token）：
```json
{"code":200,"message":"success","data":"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}
```

**后续请求需在 Header 中携带**：
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**异常场景**：
- 用户不存在 → `{"code":500,"message":"用户不存在"}`
- 密码错误 → `{"code":500,"message":"密码错误"}`
- 账户被禁用 → `{"code":500,"message":"账户已被禁用"}`

### 7.4 修改密码

```http
POST /user/password
Authorization: Bearer <token>
Content-Type: application/json

{
  "oldPassword": "123456",
  "newPassword": "654321"
}
```

**注意**：修改密码后原 Token 即失效，需重新登录。

### 7.8 管理员接口

#### 创建管理员账户

```bash
# 步骤1: 先注册一个普通用户
POST http://localhost:10010/user/register
{"username":"admin","password":"123456","nickname":"管理员"}

# 步骤2: 手动在MySQL中提升角色
UPDATE tea_user SET role = 1 WHERE username = 'admin';
```

#### 管理员登录获取Token

```http
POST /user/login
Content-Type: application/json
{"username":"admin","password":"123456"}
```

返回的 Token 中 `role=1`，可用于访问 `/admin/**` 接口。

#### 管理员查看用户列表

```http
GET /admin/user/list?page=1&size=10&keyword=test
Authorization: Bearer <admin_token>
```

支持 keyword 对 username 和 nickname 进行模糊搜索。

#### 验证权限拦截

使用普通用户 Token（role=0）访问 `/admin/**` 接口：
- 预期返回：`403 Forbidden`
