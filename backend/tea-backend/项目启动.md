# 本地开发环境启动指南

> **最后更新：** 2026-06-16

---
## 启动前需要提前下载对应工具：

## 一、基础设施启动

| 组件                 | 启动方式                              | 地址                                                                                              |
|--------------------|-----------------------------------|-------------------------------------------------------------------------------------------------|
| MySQL 8.0          | 服务/命令行                            | localhost:3306，库 `tea_platform`                                                                 |
| Redis              | `redis-server`                    | localhost:6379（无密码）                                                                             |
| Nacos              | 双击 `startup.cmd`                  | http://127.0.0.1:8848/nacos                                                                     |
| MinIO(RustFS替代)    | `双击RustFS`                        | 控制台：http://127.0.0.1:9001 / <br/>文件访问：http://127.0.0.1:9000/tea-platform/[文件名] (admin/12345678) |
| Sentinel Dashboard | 双击 `start-sentinel-dashboard.bat` | [控制台](http://localhost:8089) (sentinel/sentinel)                                                |

## 二、微服务启动顺序

```
① user-service      (20002)
② exhibition-service(20003)
③ product-service   (20004)
④ content-service   (20005)
⑤ gateway-service   (10010)   ← 最后启动
```

### IDE 中启动

每个模块的启动类：

| 模块 | 启动类路径 |
|------|-----------|
| user-service | `com.tea.user.UserApplication` |
| exhibition-service | `com.tea.exhibition.ExhibitionApplication` |
| product-service | `com.tea.product.ProductApplication` |
| content-service | `com.tea.content.ContentApplication` |
| gateway-service | `com.tea.gateway.GatewayApplication` |

### 命令行启动

```bash
# 在 tea-backend 目录下，按顺序执行：
mvn spring-boot:run -pl user-service -am
mvn spring-boot:run -pl exhibition-service -am
mvn spring-boot:run -pl product-service -am
mvn spring-boot:run -pl content-service -am
mvn spring-boot:run -pl gateway-service -am
```
