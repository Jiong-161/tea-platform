### 1.2 项目背景

中国茶文化源远流长，是中华优秀传统文化的重要组成部分。随着互联网和电子商务的快速发展，传统的茶文化传播和茶叶销售方式面临数字化转型的迫切需求。当前市场上缺乏一个集**茶文化内容传播、茶事展览活动、茶叶电商交易**于一体的综合型平台。

"茶文·茶文化品鉴购销服务平台"旨在打造一个兼具文化深度和商业价值的垂直领域数字平台，通过互联网技术连接茶文化爱好者、茶商和普通消费者，实现茶文化知识传播与茶叶商品交易的有机融合。

## 2. 项目概述

### 2.1 产品描述

**茶文·茶文化品鉴购销服务平台**是一个基于 SpringCloudAlibaba 微服务架构的全栈Web应用，分为两大子系统：

- **C端用户前台（client-web）：** 面向普通消费者和茶文化爱好者，提供茶文化文章浏览、茶事展览浏览与报名、茶叶商品浏览与购买、个人中心等一站式服务。
- **Admin管理后台（admin-web）：** 面向平台管理员，提供用户管理、内容审核、展览管理、商品与订单管理等运营管理功能。

系统采用前后端分离架构，通过 API Gateway 统一对外提供服务，使用 JWT + Redis 实现无状态认证，MinIO 对象存储管理图片等静态资源。

### 2.3 用户角色分析

系统定义两种角色，采用 0/1 整型枚举实现最小化 RBAC：

| 角色 | 角色值 | 权限范围 | 特征描述 |
|------|--------|----------|----------|
| 普通用户（USER） | role = 0 | 浏览文章/展览/商品、报名展览、购买商品、管理个人信息、管理收货地址 | 默认注册角色 |
| 管理员（ADMIN） | role = 1 | 所有普通用户权限 + 用户管理、内容管理、展览管理、商品管理、订单管理 | 需手动设置或数据库初始化 |

### 3.4 业务流程分析

#### 3.4.1 用户注册登录流程

```
[用户访问平台] → 点击"注册"
    → 填写：用户名(4-20位) + 密码(6-20位) + 昵称
    → 前端校验格式 → 后端校验用户名唯一性
    → BCrypt加密密码 → 插入用户表(role=0, status=1)
    → 返回注册成功 → 跳转登录页

[用户登录]
    → 输入用户名+密码
    → 后端查询用户 → 校验密码 → 校验账号状态(status=1)
    → 生成JWT Token(含userId, username, role) → Redis缓存(login:token → userId)
    → 返回Token → 前端存储到localStorage
```

#### 3.4.2 商品购买流程

```
[浏览商品列表] → 点击商品查看详情
    → 点击"加入购物车" → (需登录)
    → 校验商品是否存在且上架 → 校验库存
    → 购物车已有该商品？→ 累加数量 : 新增记录
    → 进入购物车 → 查看商品列表
    → 点击"下单" →
        ├── 单商品下单：直接创建订单
        └── 购物车批量下单：合并所有购物车商品为一个订单
    → 校验所有商品库存 → 计算总金额
    → 生成16位订单号 → 创建订单主记录 + 订单明细
    → 扣减库存 → 清空购物车
    → 返回下单成功
```

#### 3.4.3 展览报名流程

```
[浏览展览列表] → 点击展览查看详情
    → 点击"立即报名" → (需登录)
    → 校验是否重复报名(同一用户+同一展览)
    → 已报名 → 提示"已报名，无需重复"
    → 未报名 → 创建报名记录 → 返回报名成功
```

#### 3.4.4 管理员审核文章流程

```
[管理员登录] → 进入文章管理页面
    → 查看文章列表（支持按状态筛选待审核文章）
    → 点击审核 → 查看正文内容
    → 审核通过(status=1) / 驳回(status=2)
    → 更新文章状态 → 前端即时更新列表
```

## 4. 非功能性需求

### 4.1 性能需求

| 编号 | 指标 | 目标值 |
|------|------|--------|
| NF-PERF-001 | 页面加载时间（首屏） | ≤ 3秒（PC端，百兆带宽） |
| NF-PERF-002 | API响应时间（P95） | ≤ 500ms（简单查询），≤ 2s（复杂聚合） |
| NF-PERF-003 | 系统并发用户数 | 支持 ≥ 100 并发用户 |
| NF-PERF-004 | 数据库连接池 | 支持 ≥ 20 并发数据库连接 |
| NF-PERF-005 | 图片加载 | MinIO直链访问，CDN缓存策略 |

### 4.2 安全性需求

| 编号 | 要求 | 实现方式 |
|------|------|----------|
| NF-SEC-001 | 用户密码加密存储 | BCryptPasswordEncoder 单向哈希 |
| NF-SEC-002 | 接口认证 | JWT Token（24小时过期）+ Redis状态管理 |
| NF-SEC-003 | 管理员鉴权 | 网关层校验JWT中role=1，/admin/** 接口强制 |
| NF-SEC-004 | 跨域安全 | Gateway统一配置CORS，允许跨域访问 |
| NF-SEC-005 | 密码脱敏 | 查询用户信息时强制清除password字段 |
| NF-SEC-006 | 数据归属校验 | 购物车/地址/订单操作前校验userId归属 |
| NF-SEC-007 | 防越权 | 非管理员删除其他管理员账户操作被拦截 |
| NF-SEC-008 | JWT密钥配置化 | 从系统属性/环境变量读取，不硬编码 |

### 4.3 可用性需求

| 编号 | 要求 | 说明 |
|------|------|------|

| NF-AVAIL-002 | 微服务容错 | 首页聚合时单个服务调用失败不影响其他模块数据展示 |
| NF-AVAIL-003 | 错误处理 | 统一全局异常拦截，返回Result格式的错误信息 |
| NF-AVAIL-004 | 前端加载状态 | 所有异步请求显示加载动画/骨架屏 |
| NF-AVAIL-005 | 前端错误兜底 | 404页面、图片加载失败占位图、网络异常提示 |

## 8. 约束与假设

### 8.1 技术约束

| 编号 | 约束内容 |
|------|----------|
| C-01 | 后端必须使用 SpringCloudAlibaba 微服务架构 |
| C-02 | 前端必须使用 Vue 3 + Element Plus 技术栈 |
| C-03 | 数据库必须使用 MySQL 8.0 |
| C-04 | 服务注册与发现必须使用 Nacos |
| C-05 | 对象存储使用 MinIO（RustFS兼容S3协议） |
| C-06 | 开发语言：后端 Java 17，前端 JavaScript（Vue SFC） |
| C-07 | 认证鉴权方案：JWT + Redis + Gateway GlobalFilter |

### 8.2 业务假设

| 编号 | 假设内容 |
|------|----------|
| A-01 | 用户注册后默认状态为启用（status=1），角色为普通用户（role=0） |
| A-02 | 管理员账号通过数据库初始化或手动设置role=1创建 |
| A-03 | 订单创建后默认状态为"已支付"（status=1），简化支付流程（课程设计阶段） |
| A-04 | 商品上架条件为 status=1 或 status IS NULL（兼容历史数据） |
| A-05 | 管理员不能禁用或删除其他管理员账户 |
| A-06 | 同一用户对同一展览只能报名一次 |
| A-07 | JWT Token有效期24小时，Redis缓存同步过期 |
